# programmers

## 문제 제출 순서

양정화 > 사승한 > 차진환

## 규칙

- 공통의 주제를 선정
- 1일 1 프로그래머스 문제 풀이
- 본인이 작성한 풀이를 작성하고 공유
- 다른 사람의 풀이를 보고 회고 진행 & QA
- 기본 풀이 언어는 JavaScript 사용

## 방식

- 생성된 날짜 기준으로 README.md 통해서 문제를 확인
- 이슈 생성 후 생성된 이슈 브랜치에서 파일 생성 (ex: LV-0/../tare.js)
- 파일 본문에 풀이 작성
- PR 본문에 풀이 해석 작성 -> 리뷰 진행
- 모두 리뷰 진행이 완료되면, 제출자가 확인 후 merge
